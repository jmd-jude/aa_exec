How to Use

This guide will help you navigate the application to unlock latent value from your data assets.

Getting Started

The tool analyzes the structure of database schemas to reveal business intelligence across four dimensions:

  • What questions your data can answer
  • How to optimize existing data
  • Strategic data additions
  • Business rules encoded in your schema

To begin, you'll need a database schema in JSON format, or you can use one of our example schemas to explore the tool's capabilities.

Step-by-Step Guide

1. API & Model Selection

In the left sidebar:

  • Choose an API Provider:
    • OpenAI - Generally faster response times
    • Anthropic - May provide more nuanced analysis for complex schemas
  • Select a Model:
    • For OpenAI: gpt-4o offers the most comprehensive analysis, while gpt-4o-mini is faster
    • For Anthropic: claude-3-7-sonnet provides the most detailed analysis, while claude-3-5-sonnet offers a good balance of speed and depth

2. Select Analysis Dimensions

Choose which aspects of your schema you want to analyze:

  • Analytical Boundaries (Default) - Discover what questions can be answered with your current data model
  • Optimization Opportunities - Identify ways to extract more value from existing data
  • Strategic Expansion - Recommendations for high-value data additions
  • Business Logic - Surface implicit organizational knowledge encoded in your schema

Select at least one dimension. For your first analysis, we recommend starting with just "Analytical Boundaries" to understand your current capabilities.

3. Input Your Schema

Choose one of three ways to provide your schema:

  • Upload JSON File - Click "Browse files" to upload a schema JSON file
  • Paste JSON - Copy and paste your schema JSON directly into the text area
  • Use an Example Schema - Select from pre-built examples to explore the tool:
    • Marketing/Influencer Database
    • E-commerce Database
    • Sports Analytics Database

Your schema should follow standard JSON format with tables and fields as shown in the examples.

4. Run Analysis

Click the "Run Analysis" button to process your schema. Analysis typically takes 15-45 seconds depending on the model and complexity of your schema.

5. Review Results

Results are displayed in two tabs:

  • Analysis - The AI-generated insights about your schema, organized by the dimensions you selected
  • Raw Schema - A formatted view of your schema for reference

Use the "Download Analysis as Markdown" button to save the results for sharing or future reference.

Understanding Your Results

Each analysis section provides different strategic insights:

  • Analytical Boundaries shows specific business questions you can answer with your current data and their business value
  • Optimization Opportunities suggests derived metrics and segmentations that can be created without collecting new data
  • Strategic Expansion recommends specific data additions that would create disproportionate analytical value
  • Business Logic reveals organizational assumptions encoded in your schema

Tips for Best Results

  • Start Simple: Begin with one analysis dimension before exploring multiple dimensions
  • Compare Schemas: Run the same analysis on different schemas to understand relative capabilities
  • Iterate: Use insights from one analysis to refine your schema, then run additional analyses
  • Mix Models: Try different AI models on the same schema to get varied perspectives
  • Focus on Specifics: The most valuable insights are often the concrete, actionable recommendations

Example Use Cases

  • Data Teams: Identify optimization opportunities and data quality issues
  • Business Analysts: Discover what questions can be answered and guide stakeholders accordingly
  • Product Managers: Use strategic expansion recommendations to prioritize data collection in new features
  • Executives: Understand analytical capabilities and limitations for strategic planning

Troubleshooting

  • If you receive an error when uploading a JSON file, verify your JSON format is valid
  • For large schemas, the "Paste JSON" option may work better than file upload
  • If analysis seems incomplete, try selecting fewer analysis dimensions for more focused results
  • For extremely complex schemas, consider breaking them into logical segments for analysis
