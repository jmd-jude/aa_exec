How to Use SIG Explorer

This guide will help you navigate the application to generate valuable business use cases from your identity graph data.

Getting Started

SIG Explorer generates customized use cases based on your:

  • Business objectives
  • Available identity data
  • Activation channels
  • Identity graph schema

To begin, you can either upload your own identity graph schema in JSON format or use our example schema to explore the tool's capabilities.

Step-by-Step Guide

1. API & Model Selection

In the left sidebar:

  • Choose an API Provider:
    • OpenAI - Offers GPT-4o models
    • Anthropic - Offers Claude 3 models
  • Select a Model:
    • For OpenAI: gpt-4o provides comprehensive analysis, while gpt-4o-mini is faster
    • For Anthropic: claude-3-7-sonnet offers the most detailed analysis, while claude-3-5-sonnet balances speed and depth
  • Choose Prompt Quality:
    • Standard - Basic output with general use cases
    • Enhanced - Premium output with detailed implementation guidance, technical complexity assessments, and prioritization

2. Define Your Business Context

Select your primary business objective:

  • Customer Acquisition - Find and convert new customers
  • Conversion Optimization - Improve conversion rates in existing funnels
  • Customer Retention - Reduce churn and increase loyalty
  • Cross-sell & Upsell - Increase customer lifetime value
  • Audience Creation - Build valuable customer segments

Provide additional business context in the text area to help generate more relevant use cases.

3. Specify Available Identity Data

Select the identity data elements available in your environment:

  • Identity Types - Email, Device IDs, Cookies, MAIDs, Phone Numbers, Physical Addresses
  • Behavioral Data - Transaction History, Website Behavior, App Usage, Demographics

4. Define Additional Context

Specify important operational parameters:

  • Customer Base Size - Approximate number of customers
  • Data Freshness - How frequently identity data is updated
  • Activation Channels - Where marketing campaigns will be delivered
  • Privacy Requirements - Regulatory constraints to consider

5. Input Schema

Choose between:

  • Upload JSON Schema - Provide your custom identity graph structure
  • Use Example Schema - Explore capabilities with our pre-built example

6. Generate Use Cases

Click "Generate Use Cases" to process your inputs. Generation typically takes 15-60 seconds depending on the selected model and schema complexity.

7. Review Results

Results are displayed in two tabs:

  • Report - The generated use cases with implementation steps and expected outcomes
  • Data - The schema and context information used to generate the results

Use the "Download Report as Markdown" button to save the results for sharing with stakeholders.

Understanding Your Results

With Enhanced quality enabled, each use case includes:

  • Objective - Clear explanation of the business goal
  • Required Identity Graph Data - Specific data elements needed
  • Implementation Approach - Step-by-step instructions
  • Expected Business Impact - Quantified outcomes and timeframes
  • Technical Complexity - Implementation difficulty, dependencies, and privacy considerations

The "Quick Wins" section highlights which use cases offer the best combination of value and implementation simplicity.

Tips for Best Results

  • Be Specific: Provide detailed business context to generate more relevant use cases
  • Start with Enhanced Quality: Use enhanced prompts for more actionable results
  • Compare Scenarios: Run multiple analyses with different business objectives
  • Focus on Implementation: Pay special attention to technical complexity and dependencies
  • Start with Quick Wins: Implement the recommended high-ROI, low-complexity use cases first

Example Use Cases

  • Marketing Teams: Generate targeted campaign strategies based on identity resolution
  • Data Strategy Teams: Identify high-value applications for identity graph investments
  • Customer Success: Demonstrate concrete value propositions to clients
  • Product Managers: Prioritize identity-based features based on business impact
